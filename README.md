Trabajo Práctico Final: Creación de una Página Web Informativa sobre un País
Objetivo: Desarrollar una página web interactiva que presente información clave de un país, incluyendo detalles como el nombre, bandera, moneda, cantidad de habitantes, un mapa con la ubicación del país y un listado de sus principales atracciones turísticas. Al hacer clic en cada atracción, la página debe mostrar la ubicación de la misma en un mapa interactivo.


Contenido del País:
Nombre del País: El nombre del país debe mostrarse claramente en un encabezado principal.
Bandera: Incluir la bandera del país como una imagen visible.
Moneda: Mostrar la moneda oficial del país.
Cantidad de Habitantes: Mostrar el número aproximado de habitantes del país.
2. Mapa del País:
El mapa debe mostrar la ubicación general del país.
Utilizar una API de mapas (como Google Maps) para integrar un mapa interactivo.
El mapa debe ser navegable, permitiendo hacer zoom y moverlo libremente.
3. Atracciones Turísticas:
Incluir un listado de al menos tres atracciones turísticas del país.
Cada atracción debe estar listada con su nombre y debe ser clicable.
Al hacer clic en una atracción, el mapa debe desplazarse automáticamente a la ubicación de la atracción y mostrar un marcador.
4. Interactividad:
Al hacer clic en una atracción turística, la página debe centrar el mapa en las coordenadas correspondientes (latitud y longitud de la atracción).
Un marcador debe señalar la ubicación exacta de la atracción en el mapa.
La navegación debe ser fluida, permitiendo que el usuario explore el país y sus atracciones sin interrupciones.

Requisitos Técnicos:
1. Lenguajes y Tecnologías:
HTML5: Para la estructura de la página.
CSS3: Para el diseño y la presentación visual de la página.
JavaScript: Para la interactividad del mapa y la navegación entre las atracciones.
API de Mapas: Utilizar una API (ej. Google Maps) para el mapa interactivo.
2. Diseño:
La página debe ser responsiva, adaptándose tanto a dispositivos móviles como a pantallas más grandes.
El diseño debe ser limpio, organizado y legible, con un uso apropiado de colores y tipografía.
3. Optimización:
Las imágenes (ej. bandera) y otros recursos deben estar optimizados para reducir el tiempo de carga de la página.
El código debe estar limpio y estructurado adecuadamente.
4. Compatibilidad:
La página debe funcionar correctamente en los navegadores principales: Chrome, Firefox y Safari.
5. Seguridad:
Si se utiliza una API (como Google Maps), las claves deben ser gestionadas de manera segura, evitando exponerlas públicamente.

Criterios de Evaluación:
1. Funcionalidad:
10 puntos: La información del país (nombre, bandera, moneda, habitantes) es precisa y se muestra correctamente.
15 puntos: El mapa carga correctamente y muestra la ubicación del país de manera precisa.
15 puntos: El listado de atracciones turísticas es clicable y funcional.
20 puntos: Al hacer clic en una atracción, el mapa se desplaza correctamente y se muestra un marcador en la ubicación de la atracción.
2. Diseño y Usabilidad:
10 puntos: El diseño es responsivo y se adapta a dispositivos móviles y de escritorio.
10 puntos: La página es estética, con buena organización visual y una correcta elección de colores y tipografía.
3. Interactividad:
15 puntos: La interactividad del mapa es fluida (zoom, desplazamiento, clics en atracciones).
4. Optimización y Rendimiento:
10 puntos: La página carga de manera rápida y eficiente, con imágenes optimizadas contemplando accesibilidad
5. Compatibilidad:
5 puntos: La página es compatible con los navegadores principales.

Total de puntos: 100 puntos.
Aprobación: Mínimo 70 puntos.

Entrega del Trabajo:
Plazo de Entrega: La página web debe estar completa y entregada antes de la fecha límite 26 de Noviembre.
Formato de Entrega:
Subir todos los archivos necesarios (HTML, CSS, JS, imágenes, etc.) en una carpeta comprimida (ZIP).
En caso de utilizar una API con clave, incluir instrucciones sobre cómo configurarla correctamente para las pruebas.
